# GREEK LANGUAGE SETUP INSTRUCTIONS
=====================================

This guide provides comprehensive instructions for setting up and maintaining Greek language support for the eshop-theme WordPress theme.

## OVERVIEW
-----------

The eshop-theme includes Greek language support with the following files:
- el_GR.po - Translation source file (editable text)
- el_GR.mo - Compiled translation file (used by WordPress)
- eshop-theme.pot - Template file for creating new translations

## THEME INFORMATION
------------------

- Text Domain: eshop-theme
- Language Files Location: /languages/
- WordPress Locale Code for Greek: el_GR
- Plural Forms: nplurals=2; plural=n != 1;

## COMPILING TRANSLATIONS
-----------------------

### Method 1: Using Poedit (GUI Method)

1. **Download and Install Poedit**
   - Visit: https://poedit.net/download
   - Download and install the appropriate version for your system

2. **Open the PO File**
   - Launch Poedit
   - Open File → Open
   - Navigate to: /languages/el_GR.po
   - Click "Open"

3. **Edit Translations (if needed)**
   - Review existing translations in the main window
   - Click on any string to edit the translation
   - Make changes as needed

4. **Compile to MO File**
   - Go to File → Save
   - Poedit will automatically compile and save both .po and .mo files
   - Alternatively, go to File → Compile to MO

5. **Verify Compilation**
   - Check that el_GR.mo file exists in the /languages/ directory
   - The file should have a recent timestamp

### Method 2: Using WP-CLI (Command Line Method)

1. **Install WP-CLI** (if not already installed)
   ```bash
   curl https://raw.githubusercontent.com/wp-cli/builds/gh-pages/phar/wp-cli.phar -o wp-cli.phar
   php wp-cli.phar --info
   chmod +x wp-cli.phar
   sudo mv wp-cli.phar /usr/local/bin/wp
   ```

2. **Navigate to Theme Directory**
   ```bash
   cd /path/to/your/theme/directory
   ```

3. **Compile PO to MO**
   ```bash
   wp i18n make-mo languages/
   ```

4. **Alternative: Compile Specific File**
   ```bash
   msgfmt languages/el_GR.po -o languages/el_GR.mo
   ```

5. **Verify Compilation**
   ```bash
   ls -la languages/el_GR.mo
   ```

### Method 3: Using Loco Translate Plugin (WordPress Method)

1. **Install Loco Translate Plugin**
   - In WordPress admin, go to Plugins → Add New
   - Search for "Loco Translate"
   - Install and activate the plugin

2. **Configure Theme Translation**
   - Go to Loco Translate → Themes
   - Find "eshop-theme" in the list
   - Click on it

3. **Edit Greek Translation**
   - Click on the "Greek" language
   - Click "Edit" to open the translation editor
   - Make any necessary changes

4. **Save and Compile**
   - Click "Save" to save changes
   - Loco Translate will automatically compile the MO file
   - The compiled file will be saved to /languages/el_GR.mo

## CONFIGURING WORDPRESS FOR GREEK
----------------------------------

### Method 1: Site-Wide Language Setting

1. **Via WordPress Admin**
   - Go to Settings → General
   - Find "Site Language" dropdown
   - Select "Ελληνικά (Greek)"
   - Click "Save Changes"

2. **Via wp-config.php**
   - Add this line to your wp-config.php file:
   ```php
   define('WPLANG', 'el_GR');
   ```

### Method 2: User-Specific Language

1. **Enable User Language Selection**
   - Go to Users → Your Profile
   - Find "Language" section
   - Select "Ελληνικά (Greek)"
   - Click "Update Profile"

### Method 3: Multilingual Plugin

1. **Install a Multilingual Plugin**
   - WPML (Premium)
   - Polylang (Free/Premium)
   - Weglot (Premium)

2. **Configure Greek Language**
   - Add Greek as a language option
   - Set up language switching
   - Configure translation settings

## TROUBLESHOOTING
-----------------

### Issue: Translations Not Appearing

**Symptoms:**
- Site displays in English despite Greek configuration
- Some strings remain untranslated

**Solutions:**

1. **Check MO File Exists**
   ```bash
   ls -la languages/el_GR.mo
   ```

2. **Verify File Permissions**
   ```bash
   chmod 644 languages/el_GR.mo
   chmod 644 languages/el_GR.po
   ```

3. **Clear Caches**
   - Clear WordPress cache (if using caching plugins)
   - Clear browser cache
   - Clear server cache (if applicable)

4. **Check Text Domain Loading**
   - Verify functions.php loads text domain:
   ```php
   load_theme_textdomain('eshop-theme', get_template_directory() . '/languages');
   ```

5. **Check WordPress Language Setting**
   - Verify WPLANG is set to 'el_GR'
   - Check Settings → General in WordPress admin

### Issue: MO File Not Being Loaded

**Symptoms:**
- No translations appear
- Debug shows "MO file not found" errors

**Solutions:**

1. **Verify File Location**
   - Ensure el_GR.mo is in /languages/ directory
   - Check directory structure is correct

2. **Check File Format**
   - Ensure file is compiled correctly
   - Try recompiling with different method

3. **Verify WordPress Version**
   - Ensure WordPress supports the locale
   - Update WordPress if needed

4. **Check Server Configuration**
   - Verify PHP has file reading permissions
   - Check .htaccess rules

### Issue: Text Domain Problems

**Symptoms:**
- Only some strings are translated
- Inconsistent translation behavior

**Solutions:**

1. **Check Text Domain Consistency**
   - Ensure all translation functions use 'eshop-theme'
   - Verify no typos in text domain

2. **Check Function Usage**
   ```php
   // Correct usage
   __('String', 'eshop-theme')
   _e('String', 'eshop-theme')
   
   // Incorrect usage
   __('String')  // Missing text domain
   __('String', 'wrong-domain')  // Wrong text domain
   ```

3. **Verify Theme Loading**
   - Check that theme is properly activated
   - Verify text domain is loaded early

### Issue: File Permission Problems

**Symptoms:**
- Unable to save translation files
- MO file not accessible

**Solutions:**

1. **Check Directory Permissions**
   ```bash
   chmod 755 languages/
   chmod 644 languages/*.po
   chmod 644 languages/*.mo
   ```

2. **Check File Ownership**
   ```bash
   chown www-data:www-data languages/
   chown www-data:www-data languages/*.po
   chown www-data:www-data languages/*.mo
   ```

3. **Verify Server Configuration**
   - Check PHP open_basedir restrictions
   - Verify file system permissions

## UPDATING TRANSLATIONS
-----------------------

### When Theme Updates

1. **Backup Current Translations**
   ```bash
   cp languages/el_GR.po languages/el_GR.po.backup
   cp languages/el_GR.mo languages/el_GR.mo.backup
   ```

2. **Update POT File**
   - Use WP-CLI:
   ```bash
   wp i18n make-pot . languages/eshop-theme.pot --domain=eshop-theme
   ```

3. **Merge New Strings**
   - Open el_GR.po in Poedit
   - Go to Catalog → Update from POT file
   - Select the updated POT file
   - Review and translate new strings

4. **Recompile MO File**
   - Save the updated PO file
   - Poedit will automatically recompile the MO file

### Adding New Strings

1. **In Theme Files**
   ```php
   // Add new translatable string
   __('New string to translate', 'eshop-theme')
   
   // With variables
   sprintf(__('Welcome %s', 'eshop-theme'), $username)
   
   // Plural forms
   _n('One item', '%d items', $count, 'eshop-theme')
   ```

2. **Update POT File**
   ```bash
   wp i18n make-pot . languages/eshop-theme.pot --domain=eshop-theme
   ```

3. **Update PO File**
   - Open el_GR.po in Poedit
   - Update from POT file
   - Translate new strings
   - Save and compile

### Handling Plural Forms in Greek

Greek uses the following plural rule:
- Singular: n == 1
- Plural: n != 1

**Example in PO file:**
```
msgid "%d item"
msgid_plural "%d items"
msgstr[0] "%d προϊόν"
msgstr[1] "%d προϊόντα"
```

**In PHP code:**
```php
$translated = _n('%d item', '%d items', $count, 'eshop-theme');
echo sprintf($translated, $count);
```

## BEST PRACTICES
-----------------

1. **Regular Updates**
   - Review translations monthly
   - Update after theme updates
   - Check for new strings

2. **Quality Assurance**
   - Have native speakers review translations
   - Test translations in context

3. **Version Control**
   - Commit translation files to Git
   - Track changes over time

1. **For Developers**

   - Provide context for translators
   - Test with different languages

2. **For Translators**
   - Use consistent terminology
   - Consider cultural context
   - Test translations on live site

3. **For Site Administrators**
   - Keep translations updated
   - Monitor for missing strings
   - Test after updates
